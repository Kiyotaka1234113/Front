## 2. Конкретные промпты

### Промпт 1:
"Как правильно типизировать `RootStackParamList` в TypeScript, чтобы экран Profile принимал обязательный параметр `userId: string`, а остальные экраны были без параметров?"

**Что получил:** Пример структуры объекта, где для Profile указывается тип `{ userId: string }`, а для остальных — `undefined`. Также AI объяснил, как использовать `NativeStackScreenProps` для получения доступа к `route.params` внутри компонента.

### Промпт 2:
"Как настроить `screenOptions` в `Stack.Navigator`, чтобы заголовок (header) был синего цвета с белым жирным текстом для всех экранов сразу?"

**Что получил:** Код с использованием свойств `headerStyle` (backgroundColor), `headerTintColor` и `headerTitleStyle` (fontWeight: 'bold').

### Промпт 3:
"В чем разница между `navigation.navigate()` и `navigation.push()` в контексте стековой навигации?"

**Что получил:** Объяснение, что `navigate` переходит на существующий экран в стеке, а `push` всегда добавляет новый экземпляр экрана сверху, что важно для вложенных переходов.

## 3. Модификация и проверка
- **Типизация:** Изначально возникла ошибка при попытке прочитать `route.params` в ProfileScreen. С помощью AI исправил определение типа пропсов с `NativeStackNavigationProp` на `NativeStackScreenProps`, так как второй вариант включает в себя и объект `route`.
- **Навигация:** Добавил проверку наличия `userId` перед рендером данных профиля, чтобы избежать падения приложения при пустых параметрах.
- **Стили:** Скорректировал отступы в `SafeAreaView`, чтобы элементы интерфейса не перекрывались системной строкой состояния на iOS и Android.

### Lab 13.2: Bottom Tab Navigation

## 1. Информация об использовании AI
- **Инструмент:** Google Gemini
- **Дата:** 2026-04-11

## 2. Конкретные промпты

### Промпт 1:
"Как реализовать вложенную навигацию (Nested Navigation), чтобы каждая вкладка Bottom Tab имела свой собственный Stack Navigator?"

**Что получил:** Схему, где в качестве `component` для `Tab.Screen` передается не обычный экран, а функция-компонент, возвращающая `Stack.Navigator`. Это позволило сохранить независимую историю переходов в каждой вкладке.

### Промпт 2:
"Как создать кастомный компонент иконки для таб-бара, который умеет отображать красный кружок (badge) с числом уведомлений поверх иконки Ionicons?"

**Что получил:** Код компонента `TabIcon`, использующего `View` с `position: 'absolute'` для отрисовки бейджа и логику отображения текста "9+", если уведомлений больше девяти.

### Промпт 3:
"Ошибка: 'The action NAVIGATE with payload was not handled by any navigator'. Почему я не могу перейти из одной вкладки в стек другой вкладки?"

**Что получил:** Разъяснение о том, что при вложенной навигации нужно указывать имя навигатора и имя экрана внутри него, либо правильно настроить общие типы в `types.ts`.

## 3. Модификация и проверка
- **Интеграция поиска:** Реализовал логику фильтрации в `SearchScreen` на основе `useState`. AI помог оптимизировать поиск через `toLowerCase()`, чтобы он не зависел от регистра.
- **Бейджи:** Настроил динамическое отображение количества уведомлений в `NotificationsScreen`. При нажатии на уведомление статус `read` меняется на `true`, и счетчик в таб-баре уменьшается.
- **Стилизация:** Изменил `tabBarActiveTintColor`, чтобы активная вкладка соответствовала брендовому синему цвету приложения.
